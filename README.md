# DataStorm 7.0 — Team DataX

End-to-end pipeline for estimating **January 2026 Maximum Monthly Purchase Potential**
for 20,000 traditional-trade beverage outlets in Sri Lanka.

The problem has **no target variable**: observed monthly volume is the censored minimum of
true demand and systemic constraints (credit, stockouts, delivery caps). We build a
defensible statistical framework — peer-conditional Q90 ceiling with a hierarchical
fallback, calibrated seasonality + YoY + holiday multipliers, spatial/POI demand signals,
and independent cross-checks on censored demand. **No supervised ML training** is used:
there is no `y` and any model trained on the censored historical volume would bake in the
censoring.

## Architecture (Bronze → Silver → Gold → Model)

```
data/
├── source/      Raw competition CSVs (placed by user; NOT committed)
├── bronze/      Immutable copies + sha256 manifest (cross-file integrity check)
├── silver/
│   ├── clean/   DQ-passed datasets (parquet)
│   └── quarantine/  Rejected rows with documented `_rejection_reason`
└── gold/        Outlet-month panel + per-outlet feature frame

src/             Canonical Python pipeline
notebooks/       00_demo_end_to_end.ipynb (EDA → predictions walk-through)
outputs/         DataX_predictions.csv + audit artifacts (CSV + PNG)
report/          DataX_report.pdf (5 pages) + .docx source
docs/            genai_log.md — GenAI transparency log
```

## Prerequisites

- Python 3.10 or newer
- Git

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/AdithaBuwaneka/datastorm-v7-storming-round.git
cd datastorm-v7-storming-round
```

### 2. Create and activate a virtual environment

```bash
# Create
python -m venv .venv

# Activate (Windows PowerShell)
.venv\Scripts\activate

# Activate (bash / Linux / macOS)
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add raw competition data

Drop the six raw files from the DataStorm 7.0 Kaggle dataset into `data/source/`:

```
data/source/
├── transactions_history_final.csv
├── outlet_master.csv
├── outlet_coordinates.csv
├── distributor_seasonality_details.csv
├── holiday_list.csv
└── 1. dataset_description.xlsx
```

## Run end-to-end

```bash
python -m src.run_pipeline
```

Or run phase by phase:

```bash
python -m src.bronze_ingest      # Phase 1: copy raw + sha256 manifest
python -m src.silver_clean       # Phase 2: DQ checks + forensics + quarantine
python -m src.poi_scraper        # Phase 3: Overpass POI fetch + BallTree join
python -m src.gold_features      # Phase 4: outlet-month panel + POI/SKU/spatial features
python -m src.potential_model    # Phase 5: peer-Q90 + Phase 3 benchmark + predictions
```

The 5-page summary report (`report/DataX_report.pdf`) is included as a final deliverable
and is not regenerated by the pipeline.

Note: the Round-2 action plan calls the potential-model upgrade "Phase 3". In this codebase
that work lives in `src.potential_model` because pipeline phases are numbered by data layer.

## Methodology Summary (full detail in `report/DataX_report.pdf`)

1. **Constraint detection per outlet-month** — four-rule disjunction: stockout sandwich, zero-in-active-outlet, no-cooler high-zero, and no-cooler low-volume (operating below own Q90/3).
2. **Peer-conditional Q90 ceiling** with hierarchical fallback (`Type × Size × Province × POI-tier` → `Type × Size × Province` → `Type × Size` → `Type` → Global; min 30 unconstrained observations per cohort).
3. **Base potential** = `max(own Q90 of unconstrained months, peer-cohort Q90)`.
4. **January 2026 projection** = base × seasonality(Jan 2026 per distributor) × YoY growth × holiday-count multiplier. Seasonality is calibrated empirically from the categorical `Seasonality_Index`; YoY is geometric mean of Jan-over-Jan growth per distributor; Jan 2026 holidays are manually entered (Duruthu Poya + Tamil Thai Pongal).
5. **Sanity bounds** — floor = Q95 of own history; ceiling = 5 × peer cohort Q90.
6. **Phase 3 business-formula benchmark** — auditable ceiling formula using historical peak, outlet type/size multipliers, January seasonality, holiday multiplier, distance-decay POI lift, and nearby-competitor drag. Saved to `outputs/audit/phase3_business_formula.csv`.
7. **Cross-check #1 — log-linear regression** on the unconstrained subset using outlet attributes, POI/spatial features, and SKU-mix features (premium/mid/mass share, avg price/L). Spearman ρ with peer-Q90 = **0.864**.
8. **Cross-check #2 — Tobit Type I MLE** (right-censored at observed value when constrained). The fit uses a deterministic sample for runtime and predicts all outlets. Spearman ρ with peer-Q90 = **0.838**; with log-linear = **0.923**.
9. **Cross-check #3 — Stochastic Frontier Analysis (SFA)** (Aigner et al 1977 half-normal). Models `log V = X β + v - u`, separates noise from inefficiency, and uses deterministic MLE sampling for reproducibility. Spearman ρ with peer-Q90 = **0.853**.
10. **5-method ensemble** when all methods converge: peer-Q90 anchor + log-linear + Tobit + SFA + Phase 3 business formula. Current convergence threshold is Spearman ρ ≥ 0.75.
11. **Hold-out validation** — predict Jan 2025 from 2023+2024 only; compare to actual Jan 2025 unconstrained values. Latest artifact Spearman ρ = **0.794** over 11,256 outlets.
12. **Internal consistency** — quantile sensitivity (Q85/Q90/Q95), constraint-rate breakdown, constraint-threshold sensitivity (25/30/40/50%), magnitude ratio, spatial Spearman.
13. **Business recommendations** — Top-100 outlet ROI list ranked by predicted-vs-observed uplift × cooler-priority weight; Sri Lanka choropleth map + per-province bars for visual deployment guidance.

## Reproducibility

- Bronze sha256 manifest tracks raw-file integrity.
- All randomness seeded.
- Tobit/SFA MLE steps use deterministic row samples so full pipeline runs stay tractable while predictions are still generated for every outlet.
- Parquet for intermediate storage; final predictions stay CSV per spec.
- Overpass API responses cached to `data/bronze/poi_raw/` so re-runs are network-free.
- One config file (`src/config.py`) is the single source of truth for paths, thresholds, and POI tags.
